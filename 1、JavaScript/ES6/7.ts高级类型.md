以下精选几道典型题目，并提供答案和解析。

### 1. `Pick`（Easy）
**题目**：实现 `MyPick<T, K>`，从 T 中选取 K 属性。
```typescript
type MyPick<T, K extends keyof T> = {
  [P in K]: T[P];
};
// 测试
type Eg = MyPick<{ a: 1; b: 2 }, 'a'>; // { a: 1 }
```

### 2. `Readonly`（Easy）
**题目**：实现 `MyReadonly<T>`，将所有属性设为只读。
```typescript
type MyReadonly<T> = {
  readonly [P in keyof T]: T[P];
};
```

### 3. `TupleToObject`（Easy）
**题目**：将元组转换为对象，键/值为元组元素。
```typescript
type TupleToObject<T extends readonly (string | number | symbol)[]> = {
  [P in T[number]]: P;
};
// 测试
const tuple = ['a', 'b'] as const;
type Obj = TupleToObject<typeof tuple>; // { a: 'a'; b: 'b' }

// 备注1：typeof tuple 在 TypeScript 中，在类型注解的位置（比如 TupleToObject<...> 的尖括号里），返回的不是 JavaScript 的 'object' 字符串，而是 tuple 变量的完整类型信息。
// 备注2：T[number] 是什么含义？获取 T 中所有数字索引位置的类型的联合，得到联合类型。
// 备注3：as const加与不加，是不一样的。不加的话ts会推导出的string | number的类型，而不是具体的字面量信息。
// 没有 as const 的情况
const tuple1 = ['a', 2, 3];
type T1 = typeof tuple1;  // (string | number)[] ❌ 丢失了 'a', 2, 3

// 使用 as const 的情况
const tuple2 = ['a', 2, 3] as const;
type T2 = typeof tuple2;  // readonly ['a', 2, 3] ✅ 保留了具体值

type Eg = TupleToObject<typeof tuple>;
let obj: Eg;

// 方式1
obj['a'] = 'a';     // ✅ 可以
obj['b'] = 'b';     // ✅ 也可以！这暴露了问题

// 方式2（as const）
let obj2: Eg2;
obj2['a'] = 'a';    // ✅ 可以
obj2['b'] = 'b';    // ❌ 错误：Property 'b' does not exist
```

### 4. `First of Array`（Easy）
**题目**：获取数组第一个元素类型。
```typescript
// infer F 表示从第一个元素的位置提取出类型，命名为 F
type First<T extends any[]> = T extends [infer F, ...any[]] ? F : never;
// 测试
type A = First<[1, 2, 3]>; // 1
type B = First<[]>; // never
```

### 5. `Length of Tuple`（Easy）
**题目**：获取元组长度（不使用 `.length`）。
```typescript
// T 获取类型 T 上名为 'length' 的属性的类型。值得注意的是，<>里面的并非数组，而是作为元组类型
type Length<T extends readonly any[]> = T['length'];
// 测试
type Len = Length<[1, 2, 3]>; // 3
```

### 6. `Exclude`（Medium）
**题目**：实现 `MyExclude<T, U>`，从联合类型 T 中排除 U 中的成员。
```typescript
type MyExclude<T, U> = T extends U ? never : T;
// 测试
type Result = MyExclude<'a' | 'b' | 'c', 'a'>; // 'b' | 'c'
```
```bash
// 备注：当 T 是一个“联合类型”时，条件类型会自动遍历联合类型的每一个成员，分别应用判断。如下：
MyExclude<'a' | 'b' | 'c', 'a'>
          ↓
自动展开成：
MyExclude<'a', 'a'> | MyExclude<'b', 'a'> | MyExclude<'c', 'a'>
          ↓
分别计算：
- 'a' extends 'a' ? never : 'a'  → never
- 'b' extends 'a' ? never : 'b'  → 'b'
- 'c' extends 'a' ? never : 'c'  → 'c'
          ↓
合并结果：
never | 'b' | 'c'
          ↓
never 在联合类型中被忽略（因为 never 是空集）
          ↓
最终结果：'b' | 'c'
```

### 7. `ReturnType`（Medium）
**题目**：获取函数返回类型。
```typescript
type MyReturnType<T extends (...args: any) => any> = T extends (...args: any) => infer R ? R : never;
// 测试
type Fn = () => string;
type Res = MyReturnType<Fn>; // string
// 它通过 T extends (...args: any) => infer R 匹配(...args: any) => any，从函数类型中"抓取"返回值的类型，而忽略参数部分。
```

### 8. `Omit`（Medium）
**题目**：实现 `MyOmit<T, K>`，从 T 中排除 K 属性。
```typescript
type MyOmit<T, K extends keyof T> = {
  [P in Exclude<keyof T, K>]: T[P];
};
// 或
type MyOmit2<T, K extends keyof T> = Pick<T, Exclude<keyof T, K>>;
// 或
type MyOmit2<T, K extends keyof T> = {
    [P in T as P extends K ? never : P]: T[P]
}
// // 遍历所有属性，通过 as 动态决定是否保留，never作为属性会被剔除
```

### 9. `DeepReadonly`（Medium）
**题目**：深度只读，需要递归
```typescript
type DeepReadonly<T> = {
  readonly [P in keyof T]: T[P] extends Record<string, unknown>
    ? DeepReadonly<T[P]>
    : T[P];
};
// 注意：上面用 Record<string, unknown> 比 object 更精确（排除函数等）
```

### 10. `TupleToUnion`（Medium）
**题目**：将元组转换为联合类型。
```typescript
type TupleToUnion<T extends any[]> = T[number];
// 测试
type Union = TupleToUnion<[1, 'a', true]>; // 1 | 'a' | true
```
**发现没有，判断逻辑都是用的extends。extends 是 TypeScript 类型系统中唯一的条件判断机制。它承担了其他语言中 if、switch、模式匹配 等所有条件逻辑的功能。**

## 什么是 TypeScript 高级类型？

**高级类型（Advanced Types）** 是指 TypeScript 中那些**超越基本类型注解**（如 `string`、`number`、`boolean`、`interface` 等）的**类型操作工具和模式**。它们允许你在**类型层面**进行编程、变换、推导和组合，从而实现更精确、更灵活的类型约束。

简单说：**基础类型描述数据是什么，高级类型描述类型如何变化和组合。**高级类型看起来像是js编程，但其实是编译时就确认的类型。

## 高级类型的核心分类

### 一、类型操作符

| 操作符 | 作用 | 示例 |
|--------|------|------|
| `keyof` | 获取对象的所有键名（联合类型） | `keyof { a: 1, b: 2 }` → `'a' \| 'b'` |
| `typeof` | 获取变量的类型（在类型上下文中） | `typeof someVariable` |
| `in` | 映射类型中遍历联合类型 | `[P in "a" \| "b"]: number` |
| `as` | 类型断言（告诉编译器你比它更懂） | `value as string` |
| `is` | 类型谓词（自定义类型保护） | `pet is Fish` |
| `infer` | 在条件类型中提取部分类型 | `T extends (...args: any[]) => infer R ? R : never` |

### 二、类型工具（运算符）

| 工具 | 作用 | 示例 |
|------|------|------|
| `|` (联合类型) | 值的类型是几种之一 | `string \| number` |
| `&` (交叉类型) | 合并多个类型 | `{ name: string } & { age: number }` |
| `?` (可选) | 属性可选 | `age?: number` |
| `readonly` | 属性只读 | `readonly name: string` |
| `const` 断言 | 使字面量类型变为只读字面量 | `['a', 'b'] as const` |

### 三、条件类型 (Conditional Types)

**核心语法**：`T extends U ? X : Y`

```typescript
// 基础使用
type IsString<T> = T extends string ? true : false;
type A = IsString<'hello'>; // true
type B = IsString<number>;  // false

// 分布式条件类型（联合类型自动分发）
type ToArray<T> = T extends any ? T[] : never;
type Result = ToArray<string | number>; // string[] | number[]

// 带 infer 的条件类型
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;
type Parameters<T> = T extends (...args: infer P) => any ? P : never;
```

### 四、映射类型 (Mapped Types)

**核心语法**：`[P in KeySet]: Type`

```typescript
// 基础映射
type Readonly<T> = {
  readonly [P in keyof T]: T[P];
};

// 带修饰符的映射（+/- 添加/移除）
type Mutable<T> = {
  -readonly [P in keyof T]: T[P];  // 移除 readonly
};
type Required<T> = {
  [P in keyof T]-?: T[P];  // 移除可选
};

// 键名重映射（TS 4.1+）
type Getters<T> = {
  [P in keyof T as `get${Capitalize<string & P>}`]: () => T[P];
};
```

### 五、内置工具类型 (Utility Types)

TypeScript 官方提供了一组常用的高级类型：

| 工具类型 | 作用 | 示例结果 |
|----------|------|----------|
| `Partial<T>` | 所有属性变为可选 | `{ a?: 1; b?: 2 }` |
| `Required<T>` | 所有属性变为必选 | `{ a: 1; b: 2 }` |
| `Readonly<T>` | 所有属性只读 | `{ readonly a: 1 }` |
| `Pick<T, K>` | 挑选部分属性 | `{ a: 1 }` |
| `Omit<T, K>` | 排除部分属性 | `{ b: 2 }` |
| `Record<K, T>` | 构造键值对类型 | `{ a: 1, b: 1 }` |
| `Exclude<T, U>` | 从联合类型中排除 | `'b' \| 'c'` |
| `Extract<T, U>` | 从联合类型中提取 | `'a'` |
| `NonNullable<T>` | 排除 null/undefined | `string` |
| `ReturnType<T>` | 获取函数返回值类型 | `string` |
| `Parameters<T>` | 获取函数参数类型元组 | `[string, number]` |
| `InstanceType<T>` | 获取构造函数实例类型 | `MyClass` |

### 六、模板字面量类型 (Template Literal Types)

TS 4.1+ 引入，允许在类型层面拼接字符串。

```typescript
type Greeting = `hello ${string}`;  // "hello xxx"
type EventHandler<Ev extends string> = `on${Capitalize<Ev>}`;
// onClick, onChange, onSubmit...

type Direction = 'left' | 'right';
type MoveCommand = `move${Capitalize<Direction>}`;
// 'moveLeft' | 'moveRight'

// 结合 infer
type ExtractPath<T extends string> = T extends `/api/${infer Path}` ? Path : never;
type Path = ExtractPath<'/api/users'>; // 'users'
```

### 七、递归类型 (Recursive Types)

类型可以引用自身，用于处理嵌套结构。

```typescript
type JSONValue =
  | string
  | number
  | boolean
  | null
  | JSONValue[]
  | { [key: string]: JSONValue };

type DeepReadonly<T> = {
  readonly [P in keyof T]: T[P] extends object
    ? DeepReadonly<T[P]>
    : T[P];
};

type Path<T, K extends keyof T = keyof T> = K extends string
  ? K | `${K}.${Path<T[K]>}`
  : never;
```

### 八、类型推断 (Type Inference)

通过 `infer` 在条件类型中“提取”部分类型。

```typescript
// 提取数组元素类型
type ArrayElement<T> = T extends (infer U)[] ? U : never;

// 提取 Promise 内的类型
type UnwrapPromise<T> = T extends Promise<infer U> ? UnwrapPromise<U> : T;

// 提取函数第一个参数类型
type FirstParam<T> = T extends (arg: infer P, ...args: any[]) => any ? P : never;

// 提取字符串模式
type GetName<T extends string> = T extends `hello ${infer Name}` ? Name : never;
```

## 如何区分"基础"和"高级"？

| 维度 | 基础类型 | 高级类型 |
|------|----------|----------|
| 作用 | 描述数据的形状 | 描述类型之间的关系和变换 |
| 复杂度 | 直观、直接 | 需要逻辑推理 |
| 涉及概念 | `string`, `number`, `interface` | `infer`, `extends`, `keyof`, `映射` |
| 使用场景 | 变量注解、函数参数 | 库开发、通用工具、类型安全封装 |
| 是否可计算 | 静态声明 | 可以进行类型运算 |

## 实际应用示例

```typescript
// 场景1：从 Redux action 类型中提取 payload
type Action = 
  | { type: 'ADD'; payload: { id: number } }
  | { type: 'REMOVE'; payload: { id: number; name: string } };

type ExtractPayload<T, Type> = T extends { type: Type; payload: infer P } ? P : never;
type AddPayload = ExtractPayload<Action, 'ADD'>; // { id: number }

// 场景2：深度部分可选（支持嵌套）
type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
};

// 场景3：类型安全的 event 系统
type EventMap = {
  click: { x: number; y: number };
  change: { value: string };
};

type EventEmitter = {
  on<E extends keyof EventMap>(event: E, handler: (data: EventMap[E]) => void): void;
  emit<E extends keyof EventMap>(event: E, data: EventMap[E]): void;
};
```

掌握高级类型，意味着你不仅能**使用**类型，还能**创造和变换**类型，这是 TypeScript 进阶的关键能力。
