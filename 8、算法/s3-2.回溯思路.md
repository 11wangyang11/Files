## 回溯的“信任”方法论
与二叉树递归类似，我们把只需要关注当前层，比如全排列就是**每一层需要做的就是选择数组中任意没有被选过的元素。**，大致就是：
```js
// 伪代码
for (let i=0;i<nums.length;i++) {
    if (nums[i]未使用) {
        path.push(nums[i])
    } else {
        continue;
    }

}
```

**然后再加上回溯进入下一层。别想太多，先套用模板写一下**
如下：
```js
function permute(nums) {
    const result = [];
    const path = [];
    const used = [];

    function backtrack() {
        // 终止条件（相信到达这里时，路径已经合法且完整）
        if (path.length === nums.length) {
            result.push([...path]);
            return;
        }

        // 当前层：遍历所有可用的选择
        for (let i = 0; i < nums.length; i++) {
            if (used[i]) continue;   // 跳过已用的

            // 做选择（改变状态）
            used[i] = true;
            path.push(nums[i]);

            // 信任：这个递归调用会处理所有“剩余元素”的排列
            backtrack();

            // 撤销选择（回到当前层状态，为下一个选择做准备）
            path.pop();
            used[i] = false;
        }
    }

    backtrack();
    return result;
}
```
**回溯不是凭空出现的奇技淫巧，而是对“可变层数穷举”问题的自然解法。**上面题目我们思考循环遍历的时候，发现循环层数和nums长度一致，不是固定的2层或者3层。所以，自然就考虑回溯了。
