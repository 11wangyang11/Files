## 一、什么场景需要返回值？
必须使用返回值 + 赋值的场景：
当递归函数需要修改调用者传递进来的指针变量（即父节点中的 left/right 或链表的 next 等引用字段）的指向时。

典型例子：
1. 二叉树的插入、删除、平衡操作（可能替换整个子树）。
2. 链表的插入、删除节点（需要更新前一个节点的 next）。

不需要返回值的场景：
1. 只读取数据（如求和、查找、遍历）。
2. 只修改节点内部属性（如 node.val++，不改变 left/right 指向）。
3. 递归函数通过非返回值方式修改外部状态（如传入数组或对象包裹引用），但不推荐。

#### 记忆口诀：
改指针，必返回，外层赋；改属性，直接改，无需返。

举例，二叉搜索树的插值：
```js
// 定义二叉树节点
class TreeNode {
    constructor(val, left = null, right = null) {
        this.val = val;
        this.left = left;
        this.right = right;
    }
}

// ==================== 1. 递归插入 ====================
// 返回插入后的根节点
function insertIntoBSTRecursive(root, val) {
    // 空树，直接创建新节点作为根
    if (!root) {
        return new TreeNode(val);
    }

    // 根据 BST 性质决定插入方向
    if (val < root.val) {
        root.left = insertIntoBSTRecursive(root.left, val);
    } else if (val > root.val) {
        root.right = insertIntoBSTRecursive(root.right, val);
    }
    // 如果 val === root.val，通常 BST 不插入重复值，忽略或根据需求处理（这里忽略）
    // 如需允许重复，可将等于情况约定为插入左或右子树，但标准 BST 一般去重。
    return root;
}

// ==================== 2. 迭代插入 ====================
function insertIntoBSTIterative(root, val) {
    const newNode = new TreeNode(val);
    if (!root) {
        return newNode;
    }

    let cur = root;
    while (cur) {
        if (val < cur.val) {
            if (!cur.left) {
                cur.left = newNode;
                break;
            }
            cur = cur.left;
        } else if (val > cur.val) {
            if (!cur.right) {
                cur.right = newNode;
                break;
            }
            cur = cur.right;
        } else {
            // 值已存在，不插入（可根据需要处理，例如返回或更新）
            break;
        }
    }
    return root;
}
```

## 二、返回值的思想（对比 C++）
先看上面的例子，`root.left = insertIntoBSTRecursive(root.left, val);`，思考一下为什么需要对root.left进行赋值？因为插入操作会改变树的结构。当递归到 null 位置时，我们创建了新节点；但这个新节点必须挂到它的父节点上，否则就丢了。

核心差异：
1. C++ 引用传递：函数参数是引用的别名，函数内修改形参 = 直接修改外部变量。因此可以直接赋新节点，无需返回值。
2. JavaScript 值传递：函数参数是外部变量值的副本（对象是引用的副本）。函数内重新指向形参不影响外部变量。因此必须通过返回值 + 外部赋值来更新父节点指针。

示例对比：
```c++
// C++ 引用传递（无需返回值）
void insert(TreeNode* &root, int val) {
    if (!root) { root = new TreeNode(val); return; }
    if (val < root->val) insert(root->left, val);
    else insert(root->right, val);
}
```
```javascript
// JS 值传递（必须返回值 + 赋值）
function insert(root, val) {
    if (!root) return new TreeNode(val);
    if (val < root.val) root.left = insert(root.left, val);
    else root.right = insert(root.right, val);
    return root;
}
```
一句话总结：
返回值是为了绕过 JS 无法修改外部指针的限制。C++ 的引用传递则不需要返回值。


## 三、信任模式的思路做题（递归树/链表核心心法）
信任模式（也称为“递归黑盒思维”）：
当你调用递归函数处理一个子树时，假设这个函数已经完美地完成了任务（插入、删除、查找等），你只需要关心当前节点如何利用这个结果，而不用纠结递归内部的具体流程。

二叉搜索树插入的信任模式伪代码：
```javascript
function insert(root, val) {
    if (!root) return new TreeNode(val);          // 基终条件，创建插入的节点（因为插入节点肯定是叶子节点，所以判断!root就行）
    if (val < root.val) {                         // 插在左子树上
        root.left = insert(root.left, val);
    } else if (val > root.val) {
        root.right = insert(root.right, val);     // 插在右子树上
    }
    return root;                                  // 当前节点成为新子树根
}
```
**递归的简单心法：“我只看具体到某一个节点时的操作，不要用大脑模拟递归的过程，避免绕糊涂。只要处理好当前节点，过程交给递归就行了**
写递归函数时，你只需要明确这个函数是干什么的（统一语义，比如 trimBST(root) 返回修剪后的新根）。只考虑当前这一个节点：
1. 它该不该保留？
2. 如果需要，它的左右子树应该怎么处理？（直接调用同一个函数，因为语义一致）
3. 返回什么给上层？
剩下的交给递归去解决子问题 —— 你不需要钻进子树的细节里。

**比如上面的插值，具体到节点上，`root.left=xxx`就是把新节点插入到当前节点的左节点或者右节点。`root.left=insert(xxx)`包装一下递归函数，就变成“插入当前节点的左子树或者右子树了”。**

练习建议：
用信任模式手写一遍二叉树的前序、中序、后序遍历，再写插入与删除，建立肌肉记忆。
